Claude Code · MCPs

Give Claude Code
actual access to your stack.

MCPs are how Claude connects to your real tools — GitHub, Notion, Figma and more. Not just reading about them. Actually using them. Here's what they are and the ones I use daily.

5 min read · Intermediate

01 · What

What MCPs actually are

MCP stands for Model Context Protocol — an open standard that lets Claude connect to external services as tools it can call during a conversation. When an MCP is connected, Claude can search your Confluence docs, open a GitHub PR, read a Figma file, or update a Notion page — not just talk about how to do it.

Without MCPs, Claude knows about your tools the way a consultant does after a briefing. With MCPs, it knows the way a teammate does who has access to the same systems.

Without MCP

"You'll need to search your Confluence space manually for the API docs — I don't have access to your internal pages."

With Atlassian MCP

"Found 3 Confluence pages matching 'API authentication'. The most recent one was updated last week — want me to summarise it?"

MCPs follow a client-server model. Each MCP server exposes a set of tools. Claude calls those tools during a response the same way it calls any other tool — it just happens to go over the network to a real service.

03 · Tips

Things I wish I knew earlier

01

Start with one MCP, not all of them

Add GitHub first. Get comfortable with how Claude decides to use it. Then add the next. Configuring 6 at once and debugging why nothing works is painful.

02

Tell Claude which MCP to use when it's ambiguous

"Search our Confluence pages for X" is clearer than "find docs on X" when you have multiple connected sources.

03

Be explicit when you don't want Claude touching a live system

Claude will use available MCP tools to answer questions even when you didn't explicitly ask it to. If you ask "what issues are open?" and Jira is connected, it'll search Jira. Say so when you don't want that.

04

Connected doesn't mean always active

Just because an MCP is connected doesn't mean Claude uses it for everything. It only calls a tool when it's relevant to your request — so having 5 MCPs connected won't slow things down or cause noise.

05

/mcp is your debugger

If Claude says it can't find something you know exists, run /mcp first. Half the time the server disconnected quietly.
